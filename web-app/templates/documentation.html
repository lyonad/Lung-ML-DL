API Documentation

Complete guide to integrating LungAI's prediction service into your applications

Getting Started

Our RESTful API provides programmatic access to lung cancer risk predictions. All responses are returned in JSON format.

Base URL

http://localhost:5000/api

Format

All requests and responses use JSON format with UTF-8 encoding

Authentication

Currently no authentication required for local development

Note: This API is designed for research and educational purposes. For production use, implement proper authentication and rate limiting.

API Endpoints

POST /api/predict

Make a prediction for a single patient based on their risk factors and symptoms.

Request Parameters

Parameter Type Required Description
age integer Required Patient age (1-120)
gender string Required Gender ("M" or "F")
smoking string Required Smoking status ("1" or "2")
yellow_fingers string Required Yellow fingers ("1" or "2")
anxiety string Required Anxiety status ("1" or "2")
peer_pressure string Required Peer pressure ("1" or "2")
chronic_disease string Required Chronic disease ("1" or "2")
fatigue string Required Fatigue status ("1" or "2")
allergy string Required Allergy status ("1" or "2")
wheezing string Required Wheezing ("1" or "2")
coughing string Required Coughing ("1" or "2")
shortness_of_breath string Required Shortness of breath ("1" or "2")
swallowing_difficulty string Required Swallowing difficulty ("1" or "2")
chest_pain string Required Chest pain ("1" or "2")
For binary features: 1 = No, 2 = Yes

Code Examples

Python

import requests

# API endpoint
url = "http://localhost:5000/api/predict"

# Patient data
data = {
    "age": 65,
    "gender": "M",
    "smoking": "2",
    "yellow_fingers": "2",
    "anxiety": "1",
    "peer_pressure": "1",
    "chronic_disease": "2",
    "fatigue": "2",
    "allergy": "1",
    "wheezing": "2",
    "coughing": "2",
    "shortness_of_breath": "2",
    "swallowing_difficulty": "1",
    "chest_pain": "2"
}

# Make request
response = requests.post(url, json=data)

# Check result
if response.status_code == 200:
    result = response.json()
    print(f"Risk Level: {result['recommendation']['risk_level']}")
    print(f"Confidence: {result['recommendation']['confidence']:.1%}")
else:
    print(f"Error: {response.json()['error']}")

JavaScript

// API endpoint
const url = 'http://localhost:5000/api/predict';

// Patient data
const data = {
  age: 65,
  gender: 'M',
  smoking: '2',
  yellow_fingers: '2',
  anxiety: '1',
  peer_pressure: '1',
  chronic_disease: '2',
  fatigue: '2',
  allergy: '1',
  wheezing: '2',
  coughing: '2',
  shortness_of_breath: '2',
  swallowing_difficulty: '1',
  chest_pain: '2'
};

// Make request
fetch(url, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(data)
})
  .then(response => response.json())
  .then(result => {
    console.log('Risk Level:', result.recommendation.risk_level);
    console.log('Confidence:', result.recommendation.confidence);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Error Handling

Important: Always check the success field in responses and handle errors appropriately in your application.

Error Response Format

{
  "success": false,
  "error": "Error message describing what went wrong",
  "details": "Additional details (optional)"
}

HTTP Status Codes

Status Code Meaning Description
200 Success Request completed successfully
400 Bad Request Invalid input data or missing required fields
404 Not Found Endpoint doesn't exist
500 Internal Server Error Server-side error occurred
